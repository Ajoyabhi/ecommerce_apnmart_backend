Please develop the Checkout Page for the application with the following requirements:

1️⃣ User Authentication Requirement

Only logged-in users should be allowed to access the checkout page.

If a user attempts to proceed to checkout without logging in, they must be redirected to the login page.

Once authenticated, the selected product(s) must be associated with that specific user account in the backend.

Backend validation must ensure checkout cannot proceed without a valid session/token.

2️⃣ Product–User Association

When the user proceeds to checkout, the selected product(s) must be linked to the authenticated user in the database.

Ensure proper order creation logic where:

Order is mapped to user_id

Cart items are converted into order items

Data integrity is maintained

4️⃣ Automatic Address Detection Using Pincode API (Frontend Requirement)

Add a frontend API integration for automatic location detection based on pincode.

📌 Requirement:

When the user enters a valid Pincode, the frontend should automatically call the following API:

https://api.postalpincode.in/pincode/{PINCODE}
📌 Expected Behavior:

On entering the pincode:

Trigger API call

Fetch location details

Automatically populate:

City

State

(Country can default to India if applicable)

If the API returns invalid or no data:

Show a proper error message (e.g., "Invalid Pincode")

Prevent form submission until corrected

📌 Important Notes:

This API call should be made from the frontend only.

Add debounce logic to avoid excessive API calls.

Add loading state while fetching data.

Validate that the pincode is numeric and of valid length before triggering API.
